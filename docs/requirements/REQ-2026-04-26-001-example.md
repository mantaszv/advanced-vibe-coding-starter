# REQ-2026-04-26-001 — Pavyzdinis PRD: vartotojo profilio puslapis

> **Pastaba:** Tai pavyzdinis PRD, demonstruojantis CWK 4-stage pipeline output formatą. Realus PRD bus sugeneruotas per `/create-prd "..."` komandą.

## 1. Introduction / Overview

Pridėti vartotojo profilio puslapį, kuriame vartotojas gali peržiūrėti ir redaguoti savo asmeninius duomenis (vardas, el. paštas, profilio nuotrauka).

## 2. Goals

- Vartotojas mato savo asmens duomenis viename puslapyje
- Vartotojas gali atnaujinti vardą ir el. paštą su validacija
- Profilio nuotrauka įkeliama per Supabase Storage

## 3. User Stories

- *Kaip prisijungęs vartotojas*, noriu pamatyti savo profilio info, kad žinočiau, kokius duomenis sistema turi.
- *Kaip vartotojas*, noriu redaguoti el. paštą, kad gaučiau pranešimus į teisingą adresą.

## 4. Functional Requirements

- **FR-1:** Maršrutas `/profile` prieinamas tik prisijungusiems vartotojams (auth middleware).
- **FR-2:** Puslapyje rodomi: vardas (read-only iš Supabase auth), el. paštas (editable), profilio nuotrauka (editable).
- **FR-3:** El. pašto pakeitimas reikalauja patvirtinimo email (Supabase auth flow).
- **FR-4:** Nuotraukos įkėlimas ribojamas iki 2MB, tipai: PNG, JPEG.

## 5. Non-Goals

- Slaptažodžio keitimas (atskiras flow `/account/security`)
- Dviejų faktorių autentifikacija
- Subscription valdymas

## 6. Design Considerations

- shadcn/ui `<Card>` + `<Form>` komponentai
- Optimistic UI pdates email pakeitimui
- Toast pranešimai apie sėkmę/klaidas

## 7. Technical Considerations

- Supabase RLS politikos: vartotojas mato/keičia tik savo duomenis (`auth.uid() = user_id`)
- Server Action `updateProfile` su Zod validacija
- Storage bucket: `avatars/`, public read, authenticated write

## 8. Orchestration Hints

| Komponentas | Kas reikalinga |
|---|---|
| **MCP serveriai** | Supabase (DB queries + RLS), Context7 (Next.js Server Actions docs), Playwright (E2E) |

| **POST Guards** | `language-guard` (LT UI tekstai), `file-size-guard` (komponentų dydis), `test-coverage-guard` (≥ 80%) |

## 9. Risk Assessment

| Rizika | Tikimybė | Poveikis | Mitigacija |
|---|---|---|---|
| El. pašto pakeitimas be patvirtinimo (security gap) | Žema | Aukštas | Supabase native confirm flow + automatinis logout sender'iui |
| RLS politikos klaidos — vartotojas mato kitų duomenis | Vidutinė | Kritinis | E2E testas su 2 vartotojais; `security-guard` PRE gate |
| Nuotraukų storage bucket per atviras | Žema | Vidutinis | RLS storage politikos: write tik `auth.uid() = user_id` |
| Form'ų server action neturi rate limiting'o | Vidutinė | Vidutinis | Pridėti rate-limit middleware (10 req/min) |

## 10. Success Metrics

- 100% vartotojų gali pasiekti `/profile` po prisijungimo
- 0 RLS pažeidimų production'e per pirmus 30 dienų
- E2E testas praeina <5s
- Nuotraukos įkėlimo p95 latency <2s
