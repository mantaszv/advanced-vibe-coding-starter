# PRD: mini-CRM — B2B SaaS Lean MVP

## 1. Įvadas / Apžvalga

mini-CRM yra opinionated, greitas, paprastas CRM mažoms B2B SaaS sales komandoms.
Sprendžia problemą, kad Pipedrive/HubSpot yra per legacy ir feature-heavy 2–10 žmonių
komandoms, o Attio/Folk taikosi į indie hackers ir power users.

**Pozicionavimas:** "Greitas, paprastas CRM SaaS sales komandai. Be AI bloat, be 47
modulių. Setup per 5 min."

**Konkurencinė landscape (2026 tyrimas):**

| Tier | Pipedrive | Attio | Folk | HubSpot |
|---|---|---|---|---|
| Entry | €14 | €29 | $24 | ~$20 |
| Mid | €39 | €69 | $48 | ~$100 |
| Top | €59 | Custom | $80+ | ~$150 |

Diferenciacija prieš juos — **greitis + UX paprastumas + LT-first**.

## 2. Tikslai (measurable)

- **G1:** MVP shipped per 4–6 savaites (solo arba duo dev)
- **G2:** Onboarding nuo signup iki first deal kanban — ≤ 5 min
- **G3:** Page Time-To-Interactive (deals kanban, 100 deals) — < 1.5 s
- **G4:** Test coverage ≥ 80% paveiktiems failams; pgTAP RLS 100% tenant lentelių
- **G5:** Cross-tenant data leak — 0 incidents (CI gate ant `pg_policies`)
- **G6:** Stripe webhook idempotency — 0 duplicate subscription updates

## 3. User stories

**US-1 (Onboarding):** Kaip SaaS sales lead'as, noriu užregistruoti komandą per
magic link be CC, kad galėčiau išbandyti 14 d. nemokamai.

**US-2 (Pipeline):** Kaip SDR, noriu drag-drop'inti deal'ą tarp pipeline stage'ų,
kad atspindėtų realią būseną realiu laiku.

**US-3 (CSV migracija):** Kaip naujas user, noriu importuoti 500 contacts iš CSV su
column mapping, kad nereikėtų ranka įvedinėti.

**US-4 (Activities):** Kaip AE, noriu prie deal pridėti notes (markdown) ir tasks
(due_at), kad neprarasčiau follow-up konteksto.

**US-5 (Team invites):** Kaip workspace owner, noriu pakviesti 5 narius email'u, ir
Stripe automatiškai pakeistų subscription quantity.

**US-6 (Trial → paid):** Kaip user trial pabaigoje, noriu pereiti į paid plan be
duomenų praradimo per Stripe Checkout.

**US-7 (Settings):** Kaip workspace owner, noriu redaguoti pipeline stages (pridėti,
ištrinti, perkelti) per nustatymai/pipeline.

## 4. Funkciniai reikalavimai

### 4.1 Auth + tenancy

- **FR-1:** Sign-up per email magic link (Resend) ARBA Google OAuth
- **FR-2:** Po sign-in middleware tikrina `org_members`; jei nėra — redirect į
  `/onboarding/workspace`
- **FR-3:** Workspace creation: name, slug (auto-generated, editable), currency
  (EUR/USD); INSERT orgs + org_members(role=owner) + 5 default pipeline_stages +
  Stripe customer (no subscription)
- **FR-4:** Custom Access Token Hook užkrauna `org_id` į JWT `app_metadata`; po
  `createWorkspace` arba `acceptInvite` privaloma `supabase.auth.refreshSession()`
- **FR-5:** Invite flow per `org_invites` (token + expires_at); accept route
  `/kvietimas/[token]`

### 4.2 CRM core

- **FR-6:** Contacts CRUD su laukais: first_name, last_name, email, phone,
  job_title, company_id FK, owner_id; unique `(org_id, lower(email))` where email
  not null
- **FR-7:** Companies CRUD su laukais: name, domain, industry, size_range,
  owner_id; unique `(org_id, lower(domain))` where domain not null
- **FR-8:** Pipeline stages per-org configurable (pridėti, ištrinti, perkelti
  position); default 5: Lead → Qualified → Proposal → Negotiation → Won/Lost
- **FR-9:** Deals CRUD su laukais: title, value_cents, stage_id, contact_id,
  company_id, owner_id, expected_close_at, closed_at (auto kai stage = won/lost)
- **FR-10:** Deals kanban view (drag-drop tarp stage'ų; `@dnd-kit/core`; optimistic
  update + Server Action reconcile)
- **FR-11:** Deals table view (sortable, filterable, pagination)
- **FR-12:** Activities polymorphic (type IN note|task|meeting; markdown body;
  due_at, done_at; FK į contact ARBA deal)
- **FR-13:** Saved views — kiekvienas list (contacts, deals) palaiko URL-encoded
  filter state (V1 minimum: filter by owner, stage, search)

### 4.3 CSV import

- **FR-14:** File upload per Supabase Storage temporary bucket (1 MB max V1)
- **FR-15:** Column mapping wizard (UI mato CSV columns → siūlo DB field mapping)
- **FR-16:** Validation preview (rodyti pirmas 10 rows + klaidas) prieš batch insert
- **FR-17:** Auto-merge duplikatų (per `(org_id, lower(email))` arba `(org_id,
  lower(domain))`)

### 4.4 Billing

- **FR-18:** Stripe Checkout flow iš `/atsiskaitymas` (1 product, 2 prices: monthly
  €19/seat, annual €15/seat)
- **FR-19:** 14-d. trial be CC (`orgs.trial_ends_at` setup metu)
- **FR-20:** Stripe webhook handler (Supabase Edge Function): `checkout.session.completed`,
  `customer.subscription.updated/deleted`, `invoice.payment_failed/succeeded`
- **FR-21:** Idempotency lentelė `stripe_events_processed (event_id PK)`; payment-guard
  blokuoja deploy be `webhooks.constructEvent` signature verification
- **FR-22:** Auto-update Stripe quantity kai pridedamas/išmetamas org_member (su
  `proration_behavior: 'create_prorations'`)
- **FR-23:** Trial UX: >7 d. tylu, 3–7 d. banner, 1–3 d. CTA, ≤0 d. read-only mode,
  >30 d. po trial hard block

### 4.5 Settings

- **FR-24:** `/nustatymai` (profile, currency display)
- **FR-25:** `/nustatymai/komanda` (members list, invite form, role change, remove)
- **FR-26:** `/nustatymai/pipeline` (stages editor — pridėti, redaguoti, drag
  reorder, ištrinti su confirmation jei yra deals)
- **FR-27:** `/nustatymai/atsiskaitymas` (Stripe Customer Portal redirect)

## 6. Design considerations

### 6.1 UI guidelines

- shadcn/ui komponentai (owned code, ne dependency)
- Tailwind v4 `@theme inline` su OKLCH spalvomis
- Dark mode default + light toggle (V1.1, ne strict V1)
- Mobile responsive (kanban → vertical list mobile)
- Empty states privalomi visiems list'ams (be jų atrodo broken)

### 6.2 Onboarding checklist

Naujas org → 3 steps:
1. Pridėk pirmą contact'ą (arba "importuok CSV")
2. Sukurk pirmą deal'ą
3. Pakviesk komandą (skip jei solo)

### 6.3 Kalbos konvencija

- URL slug'ai: LT be diakritikos (`imones`, `uzduotys`, `sandoriai`)
- Komponentų pavadinimai, props, DB kolonėlės: angliškai
- Komitai: angliškai (`feat|fix|chore` prefix)

## 7. Techniniai apribojimai

### 7.1 Tech stack (lock-in)

- **Framework:** Next.js 16.2 (App Router, Server Actions stable, Turbopack stable)
- **UI:** React 19.2, Tailwind v4 (Oxide engine), shadcn/ui, `@dnd-kit/core`
- **DB + Auth:** Supabase Postgres + RLS + Auth (magic link + Google OAuth)
- **Email:** Resend (transactional only)
- **Billing:** Stripe (Checkout + Customer Portal + Subscriptions)
- **Deploy:** Vercel (Next.js) + Supabase (DB + Edge Functions Stripe webhook)
- **Validation:** zod (1:1 su DB schema)
- **Testing:** Vitest (unit) + pgTAP (RLS) + Playwright (E2E)

### 7.2 Architektūriniai sprendimai

**A. Duomenų modelis — Hybrid rigid + jsonb extension (atidėtas V2):**
Hardcoded lentelės; `custom_data jsonb` NĖRA V1, atidėtas. Trade-off: atsisakome
Attio "build-your-own-CRM" pozicionavimo; gauname MVP per 4–6 sav. ne 16.

**B. Multi-tenancy — Single DB + `org_id` + RLS (Supabase native):**
Visi tenants dalinasi schemą; izoliacija per `org_id` + RLS policies su
`(select auth.jwt()->'app_metadata'->>'org_id')`. Trade-off: prisiimame RLS bug
riziką (catastrophic if missed); mainais 10× pigesnė infra.

### 7.3 Privalomi RLS guardrail'ai

1. `org_id` JWT app_metadata (NE user_metadata — writeable iš client)
2. `force row level security` ant visų tenant lentelių
3. Performance pattern: `(select auth.jwt()->...)` SELECT wrap → initPlan caching
4. `org_members` lentelė + custom_access_token_hook setina JWT claim
5. pgTAP RLS testai CI gate; kiekviena lentelė turi 4 policies
   (SELECT/INSERT/UPDATE/DELETE) — fail jei trūksta

### 7.4 File-size apribojimai (per CLAUDE.md)

- Failas ≤ 300 LOC; funkcija ≤ 50 LOC; komponentas ≤ 200 LOC
- Pavojaus zonos: kanban (DnD logic), CSV import wizard, settings/pipeline editor

## 9. Risk Assessment (privaloma)

### 9.1 Critical paths affected

- [x] Payment/checkout flow
- [x] Webhook/callback handlers
- [x] Authentication/authorization
- [x] Data migrations (irreversible — RLS policies sunkiai grąžinamos)
- [x] Email/notification systems (Resend transactional)
- [ ] Background jobs/cron (V1 nėra)
- [x] Third-party API integrations (Stripe, Resend, Google OAuth)

### 9.2 Risk analysis per requirement

| FR # | Risk | Kas gali sulūžti | Mitigacija |
|---|---|---|---|
| FR-4 | **HIGH** | JWT refresh klaida → user mato tuščią CRM (RLS grąžina 0) | Integration test'as `createWorkspace` → `refreshSession` flow; UI banner "Reload page if data missing" |
| FR-6 iki FR-12 | **HIGH** | RLS policy bug → cross-tenant data leak | pgTAP testai 100% lentelių; CI scan `pg_policies` ant 4 policies; `force RLS` |
| FR-20 | **HIGH** | Webhook drop → subscription state divergens nuo Stripe | Idempotency lentelė + reconciliation cron (V2 add-on; V1 manual SQL) |
| FR-21 | **HIGH** | Webhook signature bypass → fake events | `webhooks.constructEvent(payload, sig, secret)` mandatory; payment-guard CI |
| FR-22 | **MED** | Race condition: 2 invites accept'inami tuo pat metu → wrong quantity | Postgres advisory lock arba `select ... for update` accept'inant |
| FR-15 | **MED** | CSV su klaidingu encoding (Windows-1257) → corrupt LT chars | Validation preview rodo encoding; force UTF-8 reconvert |
| FR-10 | **MED** | Optimistic update fail → kortelė lieka klaidingoj kolonoj | useOptimistic + Server Action error → revert + toast |
| FR-2 | **LOW** | Middleware infinite redirect loop | Eksplicit `next` param + max-age cookie |
| FR-23 | **LOW** | Trial expiration math klaida (timezone) | Visi `trial_ends_at` UTC; client display formatuoja per locale |
| FR-26 | **LOW** | Stage delete su deals → orphan deals | UI confirmation + `on delete restrict` ant `deals.stage_id` |

## 11. Atviri klausimai

- **Q1:** Org switching V2 — JWT primary org_id ribojimas. Kada pereiti į N×N
  user↔org? Spėjame: kai turėsim 100+ orgs, kurių multiple users priklauso
  daugiau nei vienam.
- **Q2:** GDPR data residency — V1 visi duomenys EU regione (Supabase Frankfurt).
  Reikia patvirtinti su pirmais klientais, ar pakanka.
- **Q3:** Backup strategija V1 — Supabase Pro automated daily. Per-tenant restore
  ne native — reikia pgdump scripto. Ar tai pakanka?
- **Q4:** Email deliverability LT regione — Resend default IP pool. Reikia
  monitoring'o pirmąjį mėnesį, ar inviteai nepatenka į spam.
- **Q5:** Free tier ar 30-d. trial extension power users — V2 sprendimas
  priklausys nuo trial conversion duomenų.
- **Q6:** Stripe Tax automatinis EU VAT — V1 pradžioje pridėti? Atsakymas TBD —
  jei pirmieji klientai LT, IVA nereikia; jei iš kart ES — taip.
- **Q7:** Multi-language UI architektūra — `next-intl` ar `next-i18next`?
  Sprendimas atidėtas iki EN versijos pradžios (V2).
