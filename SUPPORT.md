# Pagalba ir palaikymas

## Kur pradėti

Prieš kuriant issue, peržiūrėkite:

- [`README.md`](README.md) — setup, komandos, repo struktūra;
- [`docs/INDEX.md`](docs/INDEX.md) — išsamus projekto žemėlapis;
- [`CONTRIBUTING.md`](CONTRIBUTING.md) — PR ir issue taisyklės;
- [`SECURITY.md`](SECURITY.md) — saugumo raportavimas.

## Klausimai

Bendriems klausimams naudokite GitHub Discussions: https://github.com/ponasObuolys/advanced-vibe-coding-starter/discussions.

Jei Discussions nėra įjungtos, sukurkite issue tik tada, kai klausimas susijęs su konkrečiu bug’u, funkcija arba dokumentacijos trūkumu: https://github.com/ponasObuolys/advanced-vibe-coding-starter/issues.

## Bug’ai

Bug’ams naudokite GitHub issue template **Bug report**. Įtraukite reprodukcijos žingsnius, tikėtiną rezultatą, faktinį rezultatą ir aplinką.

## Funkcijų pasiūlymai

Naujoms funkcijoms naudokite GitHub issue template **Feature request**. Aiškiai aprašykite problemą, minimalų sprendimą ir sėkmės kriterijus.

## Saugumo problemos

Saugumo problemų neraportuokite viešuose issue. Vadovaukitės [`SECURITY.md`](SECURITY.md).

Kontaktas: labas@ponasobuolys.lt.
